tgqif

Introduction

TgQIF is a Telegram bot app for QIF export

Installation

Compilation

Prerequisites

  • Boost 1.70
  • C++ compiler with C++20 support (GCC 9.0+, Clang 8+)

Building

For build just use make release JOBS=3

Building with Docker

You can also build a static release binary in guaranteed working environment. To use it you should install docker into your system (for Ubuntu: sudo apt-get install docker and add your user to docker group) and then run make dockerized-build

Command line flags

  • --help print help message

Development

Language standard

Now C++20 is used to build the project and you are free to use all features provided by this language.

Exceptions

The RAII is used all over the project. So exceptions in the constructors are considered normal and you must expect that any class can throw you an exception accordingly to its documentation.

Every exception must be documented in \throws section of Doxygen doc of method.

Memory management

Automated memory management is strongly recommended and is widely used accordingly to Core Guidelines. Please Do use refcounted pointers instead of manual new -ing and delete -ing objects.

External tools usage

If functional you implement needs external tools usage you should add it to EXTERNAL_TOOLS list in CMakeLists.txt and use through tools::Toolset["tool"] from tools.hpp file.

Code style

Naming conventions

  • Class names start from Upper case and use CamelCase
  • method names start from lower case and use underscore_naming
  • member names start from _underscore and lower case and use _underscore_notation Example:
    class Class {
    int _number;
    public:
    void number_set();
    int number() { return _number; };
    }
    Trivial one-line getters and setters are allowed.

Spacing and general code style

The code style is maintained by uncrustify tool which can be applied by make stylefix from tgqif directory.

Adding output

If you want to add your own output see factory section.

After creating a class you should add it to CMakeLists.txt and set documentation link in main.cpp

Testing

Automated testing is performed using tests from test directory. When you add new code please cover is with tests by adding new test file and registering the test in CMakeLists.txt

Existing coverage can be seen at LCOV page
